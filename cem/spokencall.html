Vector-oriented reasoning

Doing Math with words

  • Singular/plural relation

    Word2Vec is able to understand the relation between plural and singular of words $$\left (\overrightarrow{cat} - \overrightarrow{cats} \right ) \approx \left (\overrightarrow{car} - \overrightarrow{cars} \right )$$

  • Distance between words

    Word2Vec can calculate the distance between words to find simulatiry $$\left \| \overrightarrow{men} - \overrightarrow{cat} \right \| = 0.7130494$$

  • answering analogy questions

    World analogies can be resolved with word2vec $$\overrightarrow{x} = \overrightarrow{men} + \left (\overrightarrow{woman} - \overrightarrow{queen} \right )$$

  • Element-wise addition

    Element-wise addition can be used to reveal relationships $$\overrightarrow{German} + \overrightarrow{airline} = \overrightarrow{Lufthansa}$$

Architcture of the Classifier

Binary classifier based on the distance of the ten top most reference answers to a prompt

  • Input : Distances between reference grammer and transcript
  • Layers: 2-Hidden Layeres
  • Outout : Binary Judgment - reject or accept
#Initializing Neural Network
classifier = Sequential()

# Adding the input layer and the first hidden layer
classifier.add(Dense(64, activation='relu', input_dim=10))
classifier.add(Dropout(0.2)) 
classifier.add(Dense(64, activation='relu'))
classifier.add(Dropout(0.2))
classifier.add(Dense(1, activation='sigmoid'))

Why we choosed polyaxon

  • Versioning and reproducibility

    Reproducible results with a built-in version control for code and experiments.

  • Hyperparameters search

    Run group of experiments in parallel and in distributed way.

  • Runs on any infrastructure

    Deploy Polyaxon in the cloud, on-premises or in hybrid environments.

  • User resources allocation

    Manage your team's resources and parallelism quotas.

  • Scalable

    Polyaxon can easily scale horizontally by adding more nodes.

  • Powerful interface

    Author jobs, experiments, and pipelines in Json, YAML, and Python.

Polyaxon Concepts

Running you Models

  • Project

    A Project in Polyaxon is very similar to a project in GitHub, it aims at organizing your efforts to solve a specific problem. A project consist of a name and a description, the code to execute, the data, and a polyaxonfile.yml.

  • Experiment

    An Experiment is the execution of your model with data and the provided parameters on the cluster. An Experiment Job is the Kubernetes pod running on the cluster for a specific experiment

  • Experiment Group

    A selection of experiments to compare. It might be the same version of your model and data with different hyper parameters based on a hyperparameters search algorithm
